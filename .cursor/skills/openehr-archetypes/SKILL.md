# openEHR Archetypes

When helping with openEHR archetypes (design, review, ADL, translation), read and apply the project guides. Use the Read tool to load the relevant file(s) from `resources/guides/archetypes/`.

Key guides:

- **principles.md** — design principles
- **rules.md** — normative rules
- **adl-syntax.md**, **adl-idioms-cheatsheet.md** — ADL syntax and idioms
- **checklist.md** — conformance checklist
- **terminology.md**, **language-standards.md**, **language-standards-nb.md** — terminology and language
- **structural-constraints.md**, **anti-patterns.md**, **reference-formatting.md** — as needed

Follow the guidance in the loaded guide(s). Cross-references use `openehr://guides/archetypes/{name}` (same as MCP).
